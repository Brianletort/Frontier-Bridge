# Running frontier MoE models on 128 GB unified-memory machines

Generated by `frontier runbook render` from `runbooks/unified-128gb.yaml` — do not edit by hand.

**Hardware class:** unified_128gb — Coherent unified-memory machines in the ~120-145 GB class: Apple M5 Max 128 GB and NVIDIA GB10 / DGX Spark. One memory pool serves CPU and GPU, so the hierarchy has two levels (unified -> SSD) instead of three — a 284B MoE fits resident; larger models depend on the SSD tier.


Committed profiles known to match: `apple_m5_max_137gb_detected`, `gb10_128gb`

## Does this runbook apply to your machine?

Run `frontier runbook match` — it profiles your machine and checks it against every runbook's requirements. This class requires:

- memory, class in unified, capacity_gb {'min': 110, 'max': 145}
- compute, class in gpu

## Bring-up

1. Install Frontier Bridge and profile the machine.

   ```bash
   git clone https://github.com/Brianletort/Frontier-Bridge.git && cd Frontier-Bridge
pip install -e . && frontier detect -o hardware_profiles/local/my_machine.yaml

   ```

   *Check:* The profile must show a single memory node with class: unified. Two memory nodes on a GB10-class machine means the GPU name missed the unified-memory markers — file an issue.


2. Measure the SSD at expert-slice granularity (this number is your streaming floor for models that do not fit resident).

   ```bash
   frontier bench ssd-stream --output results/local/ssd_expert_read.jsonl
   ```

   *Check:* Record the worst figure as measured.expert_read_gbps on the profile's storage node. Sequential bandwidth flatters the SSD; expert misses read megabytes at random offsets.


3. Plan before you download — artifacts are 107-467 GB and the planner tells you first whether your machine is worth the bandwidth.

   ```bash
   frontier plan deepseek-v4-flash --quant q2_imatrix --hardware <your profile id> --workload coding_agent --ctx 32768
   ```

## Troubleshooting

- **Planner reports far less memory than the machine was sold with.** — check: Marketing GiB vs decimal GB — a "128 GB" M5 Max detects as 137.4 decimal GB; a "128 GB" GB10 may detect lower after firmware carve-outs. The detected number is the truth the planner uses.

  Fix: None needed; trust detect.
- **Two memory nodes appear on a GB10-class machine.** — check: The GPU name did not match the unified-memory markers in detect.
  Fix: File an issue with your nvidia-smi output; the marker list is maintained in src/frontier_bridge/detect/linux_nvidia.py.
- **Decode stalls periodically on models that need the SSD tier.** — check: Expert misses are hitting storage. Compare your measured expert_read_gbps against the plan's streaming math; a slow or thermally-throttling SSD moves the worst case dramatically.

  Fix: Re-run the ssd-stream bench after sustained load to catch throttling.
